Git 是一个神奇的工具，对，神奇的！！！

至于 Git 的基本原理，可以参考上面那本 《Pro Git》，这里我们只列出安装、及基本和初步进阶的一些操作。

安装

请参考这个《Pro Git》中的 安装 Git 一节即可，不再赘述喽。 

1: jack@jk:~$ git --version
2: git version 2.17.1

配置

Git 是分布式版本控制系统，需要每台电脑自报家门：你的名字和 Email 地址。我们使用 git config 来完成这件事情，如下： 

1: jack@jk:~$ git config --global user.name "Jack Liu"
2: jack@jk:~$ git config --global user.email "loveminimal@163.com"
3: # 查询配置结果
4: jack@jk:~$ git config -l
5: user.name=Jack Liu
6: user.email=loveminimal@163.com
这里为什么要加 `--global` 呢？有没有其他的参数可用？

创建

两种创建方式：

  • 初始化本地项目
  • 直接拉取远程仓库（稍后讲述）

假设，本地存在文件夹 my-git-repo ，现在我们对它进行 Git 初始化，如下： 

1: jack@jk:~$ ls
2: Desktop  Documents  Downloads  Music  my-git-repo  Pictures  Public Videos
3: # 进入文件目录
4: jack@jk:~$ cd my-git-repo/
5: # 初始化该文件为 git 仓库
6: jack@jk:~/my-git-repo$ git init
7: Initialized empty Git repository in /home/jack/my-git-repo/.git/

^_^ 你总是可以通过 git xxx -h 去获取更多帮助。

本地仓库

git status

我们总是可以用 git status 查询当前仓库的状态，如下： 

 1: jack@jk:~/my-git-repo$ git status
 2: On branch master
 3: 
 4: No commits yet
 5: 
 6: Untracked files:
 7:   (use "git add <file>..." to include in what will be committed)
 8: 
 9:         file1
10: 
11: nothing added to commit but untracked files present (use "git add" to track)

看，它不仅给出了当前的仓库状态，还给出了你可能需要执行的操作。

git add

我们按照上面提示的操作，使用 git add 去开始追踪 file1 文件，如下： 

 1: jack@jk:~/my-git-repo$ git add file1
 2: jack@jk:~/my-git-repo$ git status
 3: On branch master
 4: 
 5: No commits yet
 6: 
 7: Changes to be committed:
 8:   (use "git rm --cached <file>..." to unstage)
 9: 
10:         new file:   file1

通过 git add ，我们做了些什么呢？

我们通过 git add file1 把文件 添加 到暂存区。

git commit

我们可以通过 git commit 命令，把暂存区中的文件 提交 到仓库。 

1: jack@jk:~/my-git-repo$ git commit -m "Commit file1."
2: [master (root-commit) c5e4830] Commit file1.
3:  1 file changed, 1 insertion(+)
4:  create mode 100644 file1
5:  # 查询一下当前的仓库状态
6: jack@jk:~/my-git-repo$ git status
7: On branch master
8: nothing to commit, working tree clean

git log

经过上边的步骤，我们已经完成了一次提交。我们可以通过 git log 去查看，我们的提交历史，如下： 

1: jack@jk:~/my-git-repo$ git log
2: commit c5e48307b1e39459287e049b84bc6f606f9dd38e (HEAD -> master)
3: Author: Jack Liu <loveminimal@163.com>
4: Date:   Mon Dec 7 17:18:15 2020 +0800
5: 
6:     Commit file1.
Git 的命令不少，常用的其实没有几个，学习它的唯一方法，就是在使用中不断的深化。
